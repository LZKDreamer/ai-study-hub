# AGENTS.md

## Project

AI Study Hub is an AI learning website. It helps users learn how AI tools solve real problems, with a secondary focus on important AI updates.

The current implementation is a Next.js data-driven static site:

- Homepage: `app/page.tsx`
- Detail article route: `app/cases/[slug]/page.tsx`
- Shared components: `components/`
- Shared styles: `app/globals.css`
- Current content data: `data/latest.json`
- Daily indexes: `data/daily/YYYY-MM-DD.json`
- Daily generation script: `scripts/generate-daily.ts`
- Candidate collection script: `scripts/collect-candidates.ts`

The original static preview files remain for reference:

- `index.html`
- `cases/codex-github-actions-pr-review/index.html`
- `styles.css`

## Product Priorities

Content priority:

1. AI latest updates that affect actual tool usage.
2. Practical cases using Codex, ChatGPT, Nano Banana, AI Agents, MCP, Skills, or plugins.
3. Tool tutorials and reusable workflows.
4. General AI news only if it has clear learning value.

Every homepage item must include:

- Content type.
- Source and time.
- Chinese title.
- Chinese summary.
- Tags.
- "为什么值得关注" for AI update items, or "解决什么问题" for case/tool items.
- "适合谁看" when applicable.
- Link to a full detail page.

## Design Rules

- Site name: `AI Study Hub`.
- Visual style: dark, compact, technical reading interface.
- Logo: simple AI tech style, not complex.
- Homepage: single-column wide cards, no right sidebar, no stats panels, no large hero.
- Detail pages: full-page articles, no left/right split layout.
- Background may use subtle stars and lightweight particle motion, but content readability comes first.
- Avoid emoji icons; use simple line-icon style SVGs or CSS shapes.
- Card radius should stay at or below 8px.

## Implementation Rules

- Keep v1 dependency-light and static-data first.
- Do not commit secrets, API keys, `.env`, generated caches, or temporary screenshots.
- Prefer static JSON/Markdown generated by scripts and deployed through Vercel.
- AI generation should be provider-adapter based. The preferred default provider is Agnes AI with an OpenAI-compatible API shape.
- Keep `data/latest.json` at exactly 20 items unless the product requirement changes.
- Keep the first 3-5 homepage items as `AI最新资讯`; practical cases/tutorials/workflows should remain the majority.
- Daily automation must be failure tolerant: if public sources, network, or AI generation fail, keep the checked-in data rather than publishing broken JSON.
- Do not hard-code secrets. Agnes AI credentials must come from `.env.local`, GitHub Secrets, or deployment environment variables.

## Current User Preferences

- AI latest updates should appear first in the list.
- Practical cases should still be the majority of content.
- Clicking the whole card should open the detail page, not only the `阅读全文` button.
- Detail pages should show article content directly.
