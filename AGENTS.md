# AGENTS.md

## Project

AI Study Hub is an AI learning recommendation website. It helps users find high-value articles, videos, and open source projects about how AI tools solve real problems.

The current implementation is a Next.js data-driven static site:

- Homepage: `app/page.tsx`
- Shared components: `components/`
- Shared styles: `app/globals.css`
- Current content data: `data/latest.json`
- Daily indexes: `data/daily/YYYY-MM-DD.json`
- Daily generation script: `scripts/generate-daily.ts`
- Candidate collection script: `scripts/collect-candidates.ts`

The original static preview files remain for reference:

- `index.html`
- `cases/codex-github-actions-pr-review/index.html`
- `styles.css`

## Product Priorities

This is not a generic AI news site. It is a curated AI learning feed. Sources are not limited to Chinese content, but displayed titles and recommendation summaries should be Chinese.

Content priority:

1. Publicly accessible articles about AI tools, real projects, AI monetization, workflows, Skills/plugins, MCP, and AI Agents.
2. YouTube videos with tutorials, demos, project walkthroughs, or tool reviews.
3. GitHub AI open source projects when public repo data is available.
4. Juejin and other technical/creator platforms about AI development, automation, prompts, plugins, and practical projects.
5. AI latest updates only when they affect real tool usage and can be explained with learning value.

Important topics include ChatGPT, Codex, Claude, Gemini, OpenAI, Seedance, 豆包, 即梦, RunningHub, libtv, MCP, Agent, Skill, plugins, AI workflows, GitHub open source projects, AI projects, and AI monetization.

Every homepage item must include:

- Author or channel name.
- Platform.
- Original article/video/project publish or update time.
- Chinese display title. Translate or adapt English titles into Chinese while preserving project/tool names.
- Chinese recommendation reason.
- 2-6 generated tags.
- Original article, video, or GitHub project link.

Do not show category chips, content type labels, "为什么值得关注", "适合谁看", or a separate "查看原文" button on the homepage.

## Recommendation Algorithm

Filter candidates by topic first, then rank by:

- Whether the content explains how to use a specific AI tool.
- Whether it includes a real project, case, workflow, result, monetization path, or useful open source implementation.
- Whether readers can reuse the method in their own work.
- Title credibility and content specificity.
- Public heat signals such as reads, views, likes, GitHub stars, and interactions when available.
- Freshness and source credibility.

Filter out:

- Pure news reposts.
- Empty opinions.
- Clickbait.
- Version-only updates.
- GitHub releases or changelogs unless a high-quality article/video explains their practical learning value.

WeChat public account content cannot be reliably searched from the web. Only process public WeChat links or public metadata that can be accessed without private WeChat search, login-only content, or anti-crawler bypasses.

AI may generate recommendation summaries and tags only. It must not invent original article content.

## Design Rules

- Site name: `AI Study Hub`.
- Subtitle: `站在AI的最前沿`.
- Main title font: `Orbitron` with hard modern sans-serif fallbacks.
- Visual style: dark, compact, technical reading interface with a space-themed background.
- Logo: transparent-background AI tech/network style, not a filled button or framed tile.
- Homepage: single-column wide cards, no right sidebar, no stats panels, no large hero.
- No secondary article pages in the current product direction; homepage cards open original sources directly.
- Background may use subtle stars, Milky Way motion, and random-direction meteors, but content readability comes first.
- Avoid emoji icons; use simple line-icon style SVGs or CSS shapes.
- Card radius should stay at or below 8px.

## Implementation Rules

- Keep v1 dependency-light and static-data first.
- Do not commit secrets, API keys, `.env`, generated caches, or temporary screenshots.
- Prefer static JSON generated by scripts and deployed through Vercel.
- AI generation should be provider-adapter based. The preferred default provider is Agnes AI with an OpenAI-compatible API shape.
- Keep `data/latest.json` at exactly 20 items unless the product requirement changes.
- If `imageUrl`, reads, views, likes, or stars are unavailable, unknown, or zero, omit the field and do not render placeholders.
- Daily automation must be failure tolerant: if public sources, network, or AI generation fail, keep the checked-in data rather than publishing broken JSON.
- Do not hard-code secrets. Agnes AI credentials must come from `.env.local`, GitHub Secrets, or deployment environment variables.

## Local Codex Skill Installation

- Install Codex skills as real directories under `C:\Users\admin\.codex\skills\<skill-name>`.
- Do not use `D:\software\Codex\home\Skill` plus Junction links for new skill installs unless explicitly requested.
- A valid skill directory must contain `SKILL.md` directly under the skill folder.

## Current User Preferences

- Public content is required; English content is allowed when the displayed title and summary are Chinese.
- WeChat public links, YouTube, Juejin, public blogs, and GitHub AI projects are priority platforms.
- AI Skill/plugin content is a key topic.
- Clicking the whole card should open the original source, not an internal detail page.
