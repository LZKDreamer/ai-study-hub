# AGENTS.md

## Project

AI Study Hub is an AI learning website. It helps the user learn how AI tools solve real problems, with a secondary focus on important AI updates.

The current implementation is a static preview:

- Homepage: `index.html`
- Detail article: `cases/codex-github-actions-pr-review/index.html`
- Shared styles: `styles.css`

## Product Priorities

Content priority:

1. AI latest updates that affect actual tool usage.
2. Practical cases using Codex, ChatGPT, Nano Banana, AI Agents, MCP, Skills, or plugins.
3. Tool tutorials and reusable workflows.
4. General AI news only if it has clear learning value.

Every homepage item must include:

- Content type.
- Source and time.
- Chinese title.
- Chinese summary.
- Tags.
- "为什么值得关注" for AI update items, or "解决什么问题" for case/tool items.
- "适合谁看" when applicable.
- Link to a full detail page.

## Design Rules

- Site name: `AI Study Hub`.
- Visual style: dark, compact, technical reading interface.
- Logo: simple AI tech style, not complex.
- Homepage: single-column wide cards, no right sidebar, no stats panels, no large hero.
- Detail pages: full-page articles, no left/right split layout.
- Background may use subtle stars and lightweight particle motion, but content readability comes first.
- Avoid emoji icons; use simple line-icon style SVGs or CSS shapes.
- Card radius should stay at or below 8px.

## Implementation Rules

- Keep the static preview clean and dependency-free unless the user explicitly asks to migrate to Next.js.
- Do not commit secrets, API keys, `.env`, generated caches, or temporary screenshots.
- If adding data automation later, prefer static JSON/Markdown generated by scripts and deployed through Vercel.
- AI generation should be provider-adapter based. The preferred default provider is Agnes AI with an OpenAI-compatible API shape.

## Current User Preferences

- The user wants AI latest updates to appear first in the list.
- Practical cases should still be the majority of content.
- Clicking the whole card should open the detail page, not only the `阅读全文` button.
- Detail pages should show article content directly.
